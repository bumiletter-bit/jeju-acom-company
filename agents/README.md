# AGENT OFFICE 실행 로직 (에이전트별 파일 분리)

에이전트 [지금 실행]이 사용하는 실행 스크립트. 파일명 = 에이전트 이름 (`세미.js` 등).

## 1차 (현재): 테스트 실행 모드

- AI API 호출 없음 — 3단계 진행상황을 2초 간격으로 재생 후 더미 결과 기록
- 단계 로그는 반드시 서버 코드(server.js의 executeAgentTestRun)가 기록 (허위 보고 방지)

## 모듈 형식

```js
module.exports = {
    steps: ['1단계 표시 문구...', '2단계...', '3단계...'], // 진행상황 라벨
    stepDelayMs: 2000,                                     // 단계 간격
    result() {                                             // 완료 시 결과 (JSONB로 저장)
        return { summary: '완료: ...', lines: ['요약 1', '요약 2', '요약 3'] };
    },
};
```

## 2차 이후: 실제 로직 교체

- 이 파일의 `result()`를 실제 DB 조회/AI 호출 로직으로 교체하면 됨 (서버 본체 수정 불필요)
- 예: 세미.js → svcGetSettlements 기반 전년 동기대비 집계
- 도구 추가 시 표준 절차: agents/tools/{도구명}.js 분리 → agent_tools 등록(시크릿은 환경변수만) → 상세 패널 도구 배지 자동 표시 → 라우팅 테이블 갱신 → 테스트 후 대표 보고
