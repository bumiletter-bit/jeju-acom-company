## 운영 프로토콜 v2 (Day 2 저녁~Day 3 확립)
1. **복명복창 유지**: 대표 요청 → 똑똑이 지시문 복창 → "진행해/ㄱ" → 등록. 승인 없이 등록 금지 (사전 승인된 판정의 사실 통보는 예외적으로 즉시 등록 가능)
2. **알림 주도 운영**: 서버가 지시함 변화를 텔레그램으로 발송 → 대표는 알림 보고 "ㄱ" → 똑똑이가 조회·3줄 요약+의견+선택지 보고
3. **검수 프로토콜 v2**: 기술 검증=클로드 코드 자체 시험 전담 / 정책·애매한 것=구현 전 대표께 질문 / 대표=스모크 1건 지정 요청 시만 실행 / 운영 중 발견=즉시 보고 루프
4. **피드백 루프 (#16 이후 목표형)**: 대표는 입력·정책 결정만 → 똑똑이가 관측 도구로 결과 직접 판독 → "정상/이상" 선보고 → 대표 만족·추가 요청 → 똑똑이 수정 지시
5. 지시 입력(마루 명령)은 대표 전용 유지 — 똑똑이 입력 도구 없음 (결재 구조). 추후 is_test 격리형 테스트 입력 도구는 아이디어로만 기록
