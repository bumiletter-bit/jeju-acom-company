## 0. 한눈에

- **무엇**: 제주아꼼이네 농업회사법인(주)의 사내 업무 프로그램. 감귤·호박 등 농산물 유통 + 오션라운지 카페 운영.

- **스택**: Node.js(Express) + PostgreSQL, 프론트는 `public/app.js`(단일 SPA성). AI = Anthropic Claude(마루·요원) + Google Gemini(미소 이미지/영상).

## 3. 마루 라우팅 (핵심 흐름)

- 대표 지시 → `pending_orders` 저장 → `processOrderWithMaru()` 가 마루(Sonnet)로 판단(route/clarify/feedback/schedule/settlement_input/multi).
- **애매하면 추측 실행 금지 → 되묻기(clarify)**. "네" 한마디로 진행되게 안내 문구 자동 부착.
- **멀티 지시**: "문구+이미지+일정"처럼 여러 건이면 subtasks로 분해해 각 요원에 **순차** 배정(같은 요원 동시배정 충돌 방지).
- **맥락 이어가기 (v5.9.13~15)**: 요원이 답한 뒤 대표가 **확인(보고 보관=run archive) 안 한 열린 상태**에서 이어 물으면, 마루가 처음부터 다시 생각하지 않고 **그 요원에게 이어 배정**. 멀티로 여러 요원이 열려 있으면 후속 내용을 보고 **맞는 요원**을 골라 그 요원 자기 스레드로 이어감. 확인 누른 뒤엔 마루가 새로 판단.
- **날짜 단일 소스**: 서버가 원문에서 날짜·요일을 확정(parseWeekdayRange 등)해 요원에 주입 — 요원 자체 계산 금지. 산출물 날짜는 코드로 대조(불일치 ⚠️ 표시).
- 오염 방어: 모델 응답의 파편 유입을 정화기로 차단.

## 4. 재무·정산 (돈 관련 — 안전망 최우선)

- **정산현황(settlement_status)** = 일자별 자금 집계(현금·미정산·거래처 입금 등 금액). 마루 `settlement_input`이 담당.
- **정산관리(settlements)** = 거래처별 품목·수량·금액(매출 원본). 화면 직접 입력 또는 **이미지 OCR 경로**.
- ⚠️ **정산현황 ≠ 정산관리** — 혼동 시 사고(7/20 "효돈 19원" 오저장). 서버 가드로 품목·수량 나열을 settlement_input에서 차단.
- **정산관리 이미지 자동입력 (v5.5~5.6)**: 발송목록 이미지 첨부 → 마루 비전(Sonnet)이 품목·박스수량 판독 → 서버가 3개 거래처(효돈농협·대성(시온)·기타거래처) 가격표와 **정확 대조**(matchSettlementItemExact) → 품목 최다 매칭으로 거래처 자동 인식 → **확인표 모달(대표 승인)** → 저장. **돈 원본이라 승인 전 저장 절대 금지.**
- **단가 매칭 주의 (v5.9.7)**: 무게 파싱은 소수점 허용(2.5kg·4.5kg 구분). 조회 시 재계산이라 저장값 무관하게 자동 정상화.
- **박스재고 [A안]**: `computeBoxStocks`가 표시 시점에 settlements를 읽어 재계산(저장 시 차감 아님). 거래처별 차감(대성→daesong, 효돈→hyodon). box_movements: 입고·시온이동·효돈이동.
- **한수 검산 (0원)**: 세미 보고 산출 직후 자동 재검산(총결제=상품+택배+이월, 거래처 합, CJ 박스×3,100). **자동 보정 절대 금지 — 오차는 있는 그대로 🧮 표시.**

  - v5.9.16: 생성 진행바(0→85%) + 완료 시 이미지 자동 표시(폴링). v5.9.17: 보고서 모달 중복 스택 수정(확인 1번에 닫힘).

## 6. 실패 수집함 (v5.9.18 — 학습 루프)

- 대표가 마루/요원 답변이 이상하면 **라이브 로그의 [❌ 실패]** 또는 **보고서의 [❌ 실패 표시]**를 누름 → `agent_feedback`(feedback_type='fail')에 **내 질문 + 요원 답변**째 적재.
- 위치: **⚙️ 관리 > 🧰 실패 수집함** (관리 칩에 실패 건수 배지).
- 항목은 질문+답변 표시, 길면 …접힘/클릭 전체보기, **항목별 🗑 삭제**(잘못 눌린 건 제거).
- **학습 방식**: 쌓이면 대표가 클로드 코드에게 "실패수집함 정리하자" → 함께 원인 분석 → 코드·프롬프트 보강. (개별 건 자동 재검증 아님 — 모아서 일괄 개선)
- 피드백 이력(칭찬·수정·코멘트)은 현재 미표시(불필요 판단).

## 7. 부가 기능

- **일정**: 마루 직접 처리(조회 즉답 / 등록 확인 1회 / 삭제 불가 안내). 휴가는 결재 시스템 안내.
- **송장변환·중간발주 (v5.9)**: 오늘 품목별 금액(pricing) 기반 특징 매칭. 거래처 필터·엑셀 거래처별 색상. 직원도 경량 카탈로그 API로 사용(단가 노출 없음).
- **텔레그램 알림**: 되묻기·완료·실패 등 행동 안내형. 토큰은 환경변수.
- **관측 MCP(똑똑이용)**: get_live_log/get_test_results/get_reports(SELECT 전용). run_capability_test·run_smoke_test는 시험 폐지로 미사용(대표 명시 요청 시만).
