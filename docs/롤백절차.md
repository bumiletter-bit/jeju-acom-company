# 롤백 절차 (0단계 버전 시스템)

> 오류 발생 시 "직전 안정 버전으로 복귀"하는 방법.
> **기본은 코드만 롤백**입니다. DB 복원은 데이터 유실 가능성이 있으므로 신중히.
> 롤백 후에는 반드시 CHANGELOG.md에 기록: `"v5.1 → v5.0 롤백, 사유: ..."`

## 1. 코드만 롤백 (기본, 권장)

명령 2개로 끝납니다. 예: v5.1에서 문제 발생 → v5.0으로 복귀

```
git push origin v5.0^{commit}:main --force
```

(`^{commit}`은 태그가 가리키는 커밋을 뜻함 — 태그를 브랜치에 바로 밀면 GitHub가 거부하므로 반드시 이 형태로. 2026-07-18 리허설에서 확인됨)

- GitHub main이 해당 태그 시점으로 되돌아가고, Render가 자동 재배포합니다.
- 화면 좌측 하단 버전 표시(또는 /api/version)로 복귀 확인.

### 다시 최신으로 복귀하려면

```
git push origin <최신 커밋 해시 또는 브랜치>:main --force
```

(로컬에 최신 커밋이 있으면 `git push origin main --force`)

## 2. 코드 + DB 롤백 (데이터까지 되돌려야 할 때만)

⚠️ 백업 시점 이후 입력된 데이터가 사라집니다. 대표 판단 후에만 실행.

1. 먼저 현재 상태를 백업 (되돌리기 전 안전장치):
   ```
   node scripts/db-backup.js 롤백전스냅샷
   ```
2. 코드 롤백 (위 1번 방법)
3. DB 복원 — CHANGELOG.md 하단 "버전-백업 매칭표"에서 해당 버전 백업 파일명 확인 후:
   ```
   node scripts/db-restore.js backup_v4.0_20260718.sql --yes
   ```
   - 복원은 트랜잭션으로 실행됨 — 중간 실패 시 전부 롤백되어 DB는 그대로 유지.

## 3. 백업 파일 위치

- `C:\Users\전승범\OneDrive\문서\제주아꼼이네_DB백업\` (리포 밖, OneDrive 동기화 = 외부 보관)
- 파일명 규칙: `backup_{버전}_{YYYYMMDD}.sql`
- 버전 ↔ 백업 파일 대응은 CHANGELOG.md 하단 매칭표가 기준.
