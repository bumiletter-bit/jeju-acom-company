# 시크릿 로테이션 절차 (v5.0 5단계 — 지시 #22)

> 원칙: 토큰·시크릿 값은 코드·로그·지시함·보고·텔레그램 어디에도 전문 기재 금지.
> 모든 시크릿은 Render 환경변수로만 관리 (.env는 git 제외, 로컬은 DATABASE_URL만 보유).

## 1. MCP URL 시크릿 (MCP_AUTH_TOKEN)

용도: `/mcp/:secret` 경로 인증 (claude.ai 커넥터 — 똑똑이). 잘못된/누락 시크릿은 404 (fail-closed).

로테이션 절차 (노출 의심 시 또는 분기 1회 권장):
1. 새 시크릿 생성: 영숫자 32자 이상 무작위 (예: PowerShell `-join ((48..57)+(97..122) | Get-Random -Count 40 | % {[char]$_})`)
2. Render 대시보드 → Environment → `MCP_AUTH_TOKEN` 값 교체 → 저장 (서비스 자동 재시작)
3. claude.ai 커넥터 설정에서 서버 URL을 새 시크릿으로 수정 → 재연결 (도구 스냅샷 갱신)
4. 확인: 구 URL 호출 404 / 새 URL `tools/list` 정상
5. 재시작으로 텔레그램 자가점검 audit(telegram_selfcheck)이 남는지 확인 (부팅 정상 지표)

## 2. 텔레그램 봇 토큰 (TELEGRAM_BOT_TOKEN)

용도: 대표 1인 수신 알림 발송. 수신처는 chat_id DB 고정 (다른 수신처 발송 경로 없음).

로테이션 절차 (2026-07-19 — 등록 과정 대화 노출 대응으로 1회 실시):
1. 텔레그램 BotFather → `/mybots` → 해당 봇 → API Token → **Revoke current token** (구 토큰 즉시 무효화)
2. 새 토큰을 Render `TELEGRAM_BOT_TOKEN` 값으로 교체 → 저장 (자동 재시작)
3. 확인: AGENT OFFICE 성장 위젯 '🔔 알림 테스트' 클릭 → 폰 수신 확인
   (재시작 후 자가점검 audit의 token_set·chat_resolved=true 확인 — chat_id는 DB에 이미 고정돼 재확보 불필요)

## 3. JWT 시크릿 (JWT_SECRET)

용도: 로그인 토큰 서명. v5.0 5단계부터 하드코딩 폴백 제거 — 미설정 시 서버 기동 실패 (fail-closed).

로테이션 절차 (노출 의심 시):
1. 새 시크릿 생성 (32자 이상 무작위)
2. Render `JWT_SECRET` 값 교체 → 저장 (자동 재시작)
3. 효과: 기존 발급 토큰 전부 즉시 무효 → 전 직원 재로그인 필요 (7일 만료보다 즉각적)
4. 확인: 재로그인 정상 + 구 토큰 401

## 점검 이력

- 2026-07-19 (v5.0 5단계): 시크릿 전문 출력 스캔 통과 — 코드·로그(console) 값 출력 0건, 요청 URL 로깅 미들웨어 없음(MCP 시크릿 비로깅), 텔레그램 발송은 tgMask(금액 가림) 경유, 프런트는 환경변수 이름만 표기. JWT 폴백 제거 적용. 텔레그램 토큰 로테이션 실시.
