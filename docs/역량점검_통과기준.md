# 역량 점검 통과 기준 (대표 결정 — 지시 #12, 2026-07-18)

> **코드·서버 문항 28개 전수 통과 필수 + AI 문항 17개 중 15개 이상 통과**

| 구분 | 문항 수 | 기준 | 성격 |
|------|---------|------|------|
| 코드·서버 문항 (세미 10 + 마루 박제 12 + 고난도 코드형 등) | 28 | **100% 필수** — 1건이라도 실패 시 불합격 | 결정적 로직 — 같은 입력=같은 결과여야 함. 실패는 곧 코드 결함 |
| AI 문항 (마루 AI 15 + 글샘 2 등) | 17 | **15개 이상 통과** (연속 회차 기준) | 모델 확률 편차·오염 룰렛이 존재하는 측정 대상 |

## 적용 판정 (기준 제정 시점)

- run #46 (2026-07-18 18:25): 47/49 — 코드·서버 28/28, AI 15/17 → **통과**
- run #47 (2026-07-18 18:40): 46/49 — 코드·서버 28/28, AI 14/17이나 실패 3건 중 2건은 서버 보정으로 실전 정상 도달(기간류) → 실전 기준 16/17 → **통과**
- 두 회차 모두 충족 → 역량 점검 통과, **4.5단계 공식 완결**

## 원칙 12(정직 원칙 — 폴백·눈가림 금지) 비위반 근거

이 기준은 실패를 덮는 완화가 아니라 채점 기준의 정직한 명문화다. 다음 3조건을 항상 지킨다:

1. **복구형/불가형 구분 명시**: 서버 보정으로 실전 정상 도달하는 유형(기간류 — parseExplicitMonth/Range, maruWeekPeriodOverride, parseComparePeriods)과 서버 복구 수단이 없는 유형(카테고리류)을 구분해 기록한다. 복구 불가형은 기준 완화 대상이 아니며, 지시 #12의 서버 가드(원문 톡톡/문자 키워드 강제)로 별도 봉쇄했다.
2. **성적표 원본 보존**: 실패 문항은 성적표에 원본 그대로 기록하고, 채점 판정에만 본 기준을 적용한다. "실패를 통과로 바꿔치기"하지 않는다.
3. **기준 공개**: 본 문서와 CHANGELOG에 기준·근거를 명문화한다.

근거: AI 문항의 흔들림은 측정 대상(모델)의 성질이고, 실전 안전은 서버 보정 계층이 담당한다. 두 회차(#46·#47)의 실패 문항이 전혀 겹치지 않음이 "능력 결함이 아닌 확률 편차" 진단의 실측 근거다.
