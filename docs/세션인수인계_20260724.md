## 1. 작업 규칙 (반드시 준수)
- **부작용 사전 고지**: 코드 수정이 다른 코드에 지장 줄 수 있으면 먼저 대표에게 알리고, 없으면 바로 진행. 일부러 꼬투리 잡지 말 것 — 진짜 위험만 정직하게. (메모리 work-rule-side-effects)
- **배포 전 로컬 검증**: AI 요원 코드는 SDK 목킹으로, 프론트/조직도는 로컬 서버 + 브라우저 스크린샷으로 배포 전 검증.
- **자동 재시도 금지**(유료 API): 과금은 대표가 직접 결정.
- 항상 한국어 응답.

## 2. 이번 세션 변경 이력 (v5.9.33 ~ v5.9.42)

### 기능/버그 수정
- **v5.9.33** LIVE 로그 정리: 실행 스텝 도배 제거(진행중=최신1줄/완료=미리보기), 지시 원문 표시(`aoCleanContent`로 `[상세조건…]`·`[멀티 x/y]` 래퍼 숨김), 흐름 한줄(🔧 마루→팀→요원), 완료 보고서에 '대표님 질문' 원본 표시.
- **v5.9.34** 🔴 이미지+질문 오류 수정: 프론트 image_data는 `data:image/png;base64,…` 전체 URL인데 마루 비전 경로가 접두사를 안 벗겨 invalid base64. 정산 OCR처럼 접두사 제거해 순수 base64 전달. + LIVE 로그 '🧹 로그 비우기' 버튼 + 오류 실행도 ✔확인 가능.
- **v5.9.35** 🔴 품목별 **결제가(단가) 이력** 조회 신규: 대표 "품목별 금액"은 매출이 아니라 **그 시기 단가**일 수 있음. `pricing`(품목별 금액 메뉴) 주간 단가 이력을 거래처별·주별 그리드로. report type `semi_price_history`. 서버 감지: `결제가|단가` 또는 (품목+주마다/주차별)이고 매출어 아님 → `price_history`. (메모리 jeju-item-price-vs-sales)
- **v5.9.37** 🔴 연속 되묻기 맥락 소실: 되묻기→"네"→또 되묻기에서 원지시가 "네"로 덮여 마루 "맥락 불명확". 되묻기 result에 `root_content`(뿌리 원지시)를 실어 체인 내내 승계.
- **v5.9.39** 🔴 글샘 '오류 마크'가 조직도에 영영 남던 버그: 완료는 idle 복귀하는데 오류는 복귀 로직 없음. ①부팅 시 running/done/error 모두 idle 초기화 ②오류도 6초 후 idle 복귀(서버·클라).
- **v5.9.41** 🔴 멀티 지시의 일정 미등록: 대표가 "네"로 전체 승인한 멀티 안의 일정 서브태스크가 각각 "이대로 등록할까요?"로 또 확인 → 순차 실행 중 서로 '대체됨'으로 밀려 미등록. **멀티 서브태스크(multiSeq)면 재확인 없이 바로 등록**(단일 등록은 확인 1회 유지). 등록 로직 `maruRegisterScheduleItems` 헬퍼로 추출.
