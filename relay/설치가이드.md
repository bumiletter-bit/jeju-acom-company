# 🛰️ 네이버 중계서버 설치 가이드 (대표님용)

> NCP 서버(101.79.16.213)에 접속해서 아래 순서대로 **복붙**하시면 됩니다.
> 리눅스 명령을 몰라도 됩니다 — 그대로 복사 → 붙여넣기 → Enter.
> 막히면 화면에 나온 **빨간 글씨/오류 메시지를 그대로** 저(클로드)에게 주세요. 바로 수정본 드립니다.

전체 그림: `[Render 회사프로그램] → [이 중계서버(고정IP)] → [네이버 커머스API]`
네이버 시크릿은 **이 서버에만** 저장됩니다(회사프로그램엔 안 들어감).

## STEP 0. 서버 접속 (NCP 방식 — 처음이면 이대로)

> NCP는 AWS와 달리 `.pem`이 **SSH 로그인 키가 아니라 'root 비밀번호를 꺼내는 열쇠'**입니다.
> 그래서 순서가: ① 방화벽 22번 열기 → ② .pem으로 root 비밀번호 확인 → ③ root + 비밀번호로 접속.

**① 방화벽(ACG) 22번 포트 열기** — NCP 콘솔 → `Server` 메뉴 → `ACG` → `akkome-vpc-default-acg` → **Inbound 규칙 추가**
- 프로토콜 `TCP` / 접근 소스 `0.0.0.0/0` / 허용 포트 `22` (유동 IP라 전체 개방 — 보안은 STEP 5에서 보완)

**② root 비밀번호 확인** — `Server` → `akkome-relay` 선택 → 상단 **`서버 관리 및 설정 변경` → `관리자 비밀번호 확인`** → **`akkome-key.pem` 파일 첨부** → `비밀번호 확인` → 나온 **비밀번호를 복사**(메모장에 보관)

**③ 접속** — 내 PC(윈도우)에서 **PowerShell** 열고:
```powershell
ssh root@101.79.16.213
```
- `Are you sure...(yes/no)` 나오면 `yes` Enter
- `root@101.79.16.213's password:` 나오면 **②에서 복사한 비밀번호 붙여넣기** (화면에 안 보여도 정상) → Enter
- `root@akkome-relay:~#` 처럼 바뀌면 접속 성공 ✅

> 잘 안 되면: NCP 콘솔의 `서버 접속 콘솔`(웹 화면)으로도 들어갈 수 있지만, 복붙이 불편하니 가급적 위 SSH를 권장.

## STEP 1. 설치 (한 줄 복붙)

아래 **한 줄**을 복사해 붙여넣고 Enter:

```bash
curl -fsSL https://raw.githubusercontent.com/bumiletter-bit/jeju-acom-company/main/relay/install.sh | sudo bash
```

이 한 줄이 자동으로: Node.js 설치 → 중계서버 코드 내려받기 → 자동실행 등록 → 시작 까지 다 합니다.
끝나면 **`🔑 RELAY_AUTH_TOKEN = xxxxx...`** 같은 값이 화면에 나옵니다.
**이 값을 메모장에 복사해두세요** (2단계에서 회사프로그램에 넣습니다).

## STEP 2. 네이버 값 채우기

아직 네이버 ID/시크릿이 비어 있습니다. 아래를 붙여넣어 편집 화면을 엽니다:

```bash
sudo nano /opt/akkome-relay/.env
```

`여기에_..._붙여넣기` 라고 된 부분을 **실제 값으로** 바꾸세요:
- `NAVER_CLIENT_ID` = 네이버 애플리케이션 ID
- `NAVER_CLIENT_SECRET` = 애플리케이션 시크릿
- (`RELAY_AUTH_TOKEN`은 자동 생성돼 있으니 **그대로 두세요**)
- (텔레그램은 이 서버에 **넣지 않습니다** — 알림은 회사프로그램이 담당하도록 설계 변경됨)

다 고쳤으면 저장하고 나가기: **`Ctrl + O`** → **Enter** → **`Ctrl + X`**

그리고 재시작:

```bash
sudo systemctl restart akkome-relay
```

## STEP 3. NCP 방화벽(ACG) 열기 — 콘솔에서 클릭

NCP 콘솔 → **ACG** → `akkome-vpc-default-acg` → **Inbound 규칙**에 아래 3개를 추가:

| 프로토콜 | 접근 소스 | 허용 포트 | 용도 |
|---|---|---|---|
| TCP | `0.0.0.0/0` | `22` | SSH 관리 (STEP 0에서 이미 열었으면 생략) |
| TCP | `74.220.52.0/24` | `4000` | Render 회사프로그램 → 중계서버 |
| TCP | `74.220.60.0/24` | `4000` | Render 회사프로그램 → 중계서버 |

> 4000 포트를 `0.0.0.0/0`(전체)로 열지 **마세요** — Render 대역 2개만 열면 됩니다.

## STEP 4. 정상 동작 확인 ⭐

서버 안에서 아래를 순서대로 붙여넣으세요.

**① 서버 살아있나:**
```bash
curl -s localhost:4000/health
```
→ `{"ok":true, ...}` 나오면 정상.

**② 네이버 토큰 발급 되나 (가장 중요):**
```bash
curl -s 'localhost:4000/health?token=1'
```
→ **`"token_test":"success"`** 가 나오면 **네이버 인증 성공!** 🎉
→ `"token_test":"fail"` 이면 나온 내용을 저에게 그대로 주세요(시크릿/IP 원인 파악).

이 결과(①②)를 저에게 알려주시면 2단계(회사프로그램 연결)로 넘어갑니다.

## STEP 5. SSH 보안 강화 (NCP 방식 — fail2ban으로 무차별 대입 차단)

> ⚠️ **주의:** NCP는 `root + 비밀번호`로 접속합니다. **비밀번호 로그인을 끄면 서버가 잠깁니다.**
> 그래서 여기서는 비밀번호 로그인을 끄지 않고, **자동 침입차단(fail2ban)**을 켭니다.
> (유동 IP라 22번을 특정 IP로 못 막으므로, 무차별 대입을 자동 차단하는 게 가장 현실적)

아래를 통째로 복붙:

```bash
sudo bash -c '
apt-get update -y
apt-get install -y fail2ban
cat > /etc/fail2ban/jail.d/sshd.local <<CONF
[sshd]
enabled = true
port = 22
maxretry = 5
findtime = 600
bantime = 3600
CONF
systemctl enable fail2ban
systemctl restart fail2ban
echo "적용 결과 (아래에 sshd가 보이면 정상):"
fail2ban-client status | grep -i jail
'
```

→ `Jail list: sshd` 가 보이면 완료. 이제 비밀번호를 5번 틀리면 그 IP가 1시간 자동 차단됩니다.

> (선택·나중에) 더 강한 보안을 원하면 → 별도 SSH 키를 서버에 등록하고 그 다음 비밀번호 로그인을 끄는 방법이 있습니다. 잠금 위험이 있어 **연결·연동이 다 안정된 뒤에** 저와 함께 안전하게 진행하겠습니다.
